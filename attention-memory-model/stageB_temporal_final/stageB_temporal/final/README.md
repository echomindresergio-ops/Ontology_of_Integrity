# stageB_temporal/final — README

## Що тут є
Фінальні результати гілки "temporal clustering" (архітектурна зміна
candidate-порівняння для вирішення winner-take-all фрагментації на
мультипікових конфігураціях сигналу). Деталі — у `report.md`.

## Структура
- `report.md` — повний технічний звіт (мета, механізм, ітерації методу,
  результати, чутливість до seed, рекомендації).
- `metrics_all_seeds.csv` — run-level метрики для seed 17, 19 (перший раунд,
  з повним per-cycle логуванням).
- `metrics_additional_5seeds.csv` — run-level метрики для seed 7, 11, 13,
  23, 29 (друга хвиля, лише зведені метрики).
- `metrics_all_7seeds.csv` — об'єднання обох файлів вище.
- `seed_sensitivity_summary.csv` — одна строка на seed: worst_run_FPR,
  worst_run_TPR(noise=1.0). Використано для розрахунку mean/95% CI в
  `report.md` §4.1.
- `seed_17/`, `seed_19/` — детальні дані лише для цих двох seed
  (першими виявлені проблемними): per-cycle агентні логи (dominant
  cluster center/points/mass, chosen candidate) для ОДНОГО представницького
  прогону кожен, і сирі attention-траси (20 циклів, temp=1.0).

## Репрезентативність даних (важливо для читання)
- **Повні (per-cycle) дані:** лише seed=17, seed=19, лише 1 representative
  run кожен (не всі N=150 прогонів), лише temp=1.0/noise=1.0 (найважчий
  випадок).
- **Лише зведені (run-level) дані:** seed 7, 11, 13, 23, 29 — немає
  per-cycle логів чи attention traces для цих seed у цьому архіві.
- **N для всіх прогонів у цій гілці:** N_null=250, N_signal=150 (НЕ
  N=1000/500, як у фінальному підтвердженні top1_vk4-конфігурації в
  попередньому раунді проєкту) -- це розвідувальний, не підтверджувальний
  рівень точності. Перед пре-реєстрацією цієї архітектурної гілки варто
  повторити ключові точки з більшим N.

## Відомі методологічні застереження
1. **FPR "по seed_templates" не є справжньою seed-залежністю.**
   Null-сценарій використовує `clean=0` незалежно від seed_templates —
   спостережений розкид worst_run_FPR (0.048-0.080 по 7 seed) відображає
   шум вибірки при N=250, а не 7 різних істинних значень. Не варто
   інтерпретувати "seed=23 гірший за seed=13 по FPR" як змістовний
   висновок.
2. **TPR-залежність від seed_templates ІСТИННА** (сигнал справді залежить
   від конфігурації прихованих піків) — тут різниця між seed=17 (0.827)
   і рештою (0.99-1.0) відображає реальну властивість методу, не шум.
3. Жодна конфігурація тут НЕ пройшла повний цикл підтвердження, який
   пройшла top1_vk4 (5 незалежних калібрувань eps, N=1000/500
   підтверджувальний прогін) — ця гілка на стадії "суттєвий, але
   неповний прогрес", не готова до фінальної пре-реєстрації без
   додаткової роботи.
